Dieses Projekt ist zum Testen von Git Merge Konflikten erstellt.

Git Commands in der richtigen Reihenfolge:

  1. git status - Zeigt aktuelle Änderungen
  2. git add . - Staged alle Änderungen
  3. git commit -m "message" - Erstellt Commit
  4. git branch feature-branch - Erstellt neuen Branch
  5. git checkout feature-branch - Wechselt zu Branch
  6. git merge main - Merged main in current branch
  7. git push origin branch-name - Pushed Branch

Merge Konflikte lösen:

  • git status - Zeigt konfliktreiche Dateien
  • Öffne die Datei und löse die Konflikte
  • git add dateiname - Markiert Konflikt als gelöst
  • git commit - Schließt Merge ab
